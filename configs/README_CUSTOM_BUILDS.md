# Создание Произвольных Билдов в Single Mode

## Использование Custom Билдов

Режим `single` теперь поддерживает создание bilдов с произвольными названиями и пользовательскими статами.

### Формат конфигурации

```json
{
  "fighter_a": {
    "type": "custom",
    "role": "Любое_Название_Билда",
    "stats": {
      "hp": 8,
      "attack": 15,
      "defense": 6,
      "agility": 11
    }
  }
}
```

### Типы билдов

1. **"balanced"** - использует предустановленные роли (ASSASSIN, TANK, etc.)
2. **"random"** - случайные статы с предустановленными ролями
3. **"custom"** - полная свобода названий и статов ✨

### Ограничения для Custom билдов

- **Каждый стат**: 3-18 (минимум 3, максимум 18)
- **Общий бюджет статов**: должен соответствовать уровню
  - Уровень 1: 17 статов (3+3+3+3 + 5)
  - Уровень 5: 37 статов (3+3+3+3 + 25)
  - Уровень 10: 57 статов (3+3+3+3 + 45)
  - Формула: 12 + (level-1) * 5

### Примеры названий билдов

```json
"role": "Атакующий_Билд_AGI_ATK"
"role": "Защитный_Билд_HP_DEF"
"role": "Критический_Убийца_v2.1"
"role": "Неубиваемый_Танк_2024"
"role": "My_Super_Build_123"
"role": "Экспериментальный_Билд_🔥"
```

### Готовые примеры

- `configs/default_single.json` - базовый пример custom билдов
- `configs/example_custom_single.json` - экстремальные билды
- `configs/default_single_custom.json` - демонстрация русских названий

### Запуск

```bash
# Использовать дефолтный конфиг
python3 main.py single

# Использовать свой конфиг
python3 main.py single configs/my_custom.json
```

### Преимущества Custom билдов

✅ **Произвольные названия** - назовите билд как угодно
✅ **Полный контроль статов** - точная настройка билда
✅ **Тестирование теорий** - проверка конкретных стат-комбинаций
✅ **Многоязычность** - поддержка русских и других символов
✅ **Версионирование** - можете включать версии в названия

Система автоматически покажет ваши custom названия в логах боя и статистике!
